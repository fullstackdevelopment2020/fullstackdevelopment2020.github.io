Project: Accessing a MS SQL Server database from an ASP.NET Core 3.0 application.


  • Let's create a MS SQL Database named Bizness and a table named Customers. The fields should be CustomerID, FirstName, LastName, Email. Don't forget that the CustomerID field is a primary key and an identifier(1,1). The other fields are just nvarchar(50).

  • Create a new ASP.NET Core application (MVC) named whatever you want.

  • Now go to Tools -> Nugget Package Manager -> Package Manager Console and type:

  • install-package microsoft.entityframeworkcore

  • install-package microsoft.entityframeworkcore.design

  • install-package microsoft.entityframeworkcore.tools

  • install-package microsoft.entityframeworkcore.sqlserver

  • And now we have to create the database context and entity classes, that is, the EF Model.

  • scaffold-dbcontext "server=your_computer_name;database=Bizness;Integrated Security=True" microsoft.entityframeworkcore.sqlserver -OutputDir Models

  • Now go to Build -> Build Solution

  • And now, create the Customers controller. Controllers -> Add -> Controller. Choose MVC controller with views using EF. Model Class: Customers and Data Context Class:BiznessContext

    .

  • Now add this line to the Startup.cs file:

  • services.AddDbContext<Models.BiznessContext>();

  • And in the same Startup.cs file, change the default controller from Home to Customers.

  • Finally press F5.

  • Add a new record.

  • Did it work? Lots of things to do before making it work. I still like .NET Framework programming better, but let's face it, the future is on .NET Core.
